Platform Operations

Understanding the simulation loop and decision-making pipeline step-by-step.

1

View Live City State

Operators monitors current Bengaluru traffic conditions on the high-fidelity interactive map, observing baseline speeds, active bottlenecks, and congestion profiles on major arterial corridors.

2

Create Scenario

Input incident parameters when a traffic disruption occurs. Specify details including location, time of day, cause type (e.g., breakdown, accident, flooding), and manual road actions (such as full or partial closures and barricades).

3

Run Simulation

Trigger the Unified Simulation Engine. The system processes the event features through trained models to estimate duration, impact severity, and runs the PyTorch ST-GNN to map congestion propagation at 15, 30, and 60-minute intervals.

4

Test Actions

Evaluate response strategies. Compare alternative diversion paths and run queue estimates to determine which options minimize delay increases, local bottlenecks, and affected area radius.

5

Deploy Plan

Review recommendations and generated command summaries. Dispatch traffic patrols, set physical diversions, and trigger adaptive signal timing overrides at nearby intersections using the optimized plans.
